01

Optimization Performance — React Native

Triết lý cốt lõi: càng ít re-render, càng mượt. Mọi kỹ thuật dưới đây đều phục vụ một mục tiêu — giữ JS Thread rảnh và UI Thread không bị nghẽn.

Memoization useCallback / useMemo / React.memo

Tránh re-render thừa bằng shallow comparison, kết hợp tách nhỏ component và custom comparison function cho các props phức tạp.

Counter.tsx
// chỉ re-render khi count thay đổi
const Counter = React.memo(({ count, user }) => {
  // ...
}, (prev, next) => prev.count === next.count);

List Rendering FlatList & FlashList

Tận dụng virtualization — chỉ render phần tử trong viewport + buffer.

  • initialNumToRender, maxToRenderPerBatch, windowSize để kiểm soát batch render
  • getItemLayout giúp tránh đo lại kích thước item
  • FlashList dùng estimatedItemSize sát giá trị thật để tối ưu lần render đầu

Threading Animated & UI Thread

Dùng Animated với useNativeDriver: true để animation chạy trực tiếp trên UI Thread, không phụ thuộc JS Thread — tránh giật lag khi JS bận.

Async InteractionManager & Lazy Load

Trì hoãn các tác vụ nặng đến khi UI mượt xong, kết hợp Suspense để lazy-load màn hình.

deferred-task.ts
InteractionManager.runAfterInteractions(() => {
  // chạy sau khi UI mượt, không chặn interaction
});

Assets Resize ảnh & định dạng tối ưu

Ưu tiên expo-image / FastImage, tránh ảnh HD không cần thiết, chọn PNG/WebP thay JPEG để giảm bộ nhớ & thời gian decode.

State Hạn chế state không cần thiết

Dùng useRef / useSharedValue cho các giá trị không cần re-render UI, giảm số lần component cập nhật.

02

Security — Banking & Fintech grade

5 lớp bảo mật áp dụng trong các ứng dụng Banking (MB Bank, LP Bank) & Fintech — từ mã nguồn đến người dùng cuối.

Lớp bảo mậtGiải pháp triển khai
1. Bảo vệ mã nguồn Hermes, ProGuard / R8, Bitcode, metro-babel obfuscation. Không hardcode API Key.
2. Bảo vệ dữ liệu Không dùng AsyncStorage cho dữ liệu nhạy cảm → Keychain / Keystore, react-native-secure-key-store, crypto-js
3. Bảo vệ giao tiếp mạng HTTPS, SSL Pinning, JWT, OAuth, mã hóa AES/RSA
4. Chống reverse engineering Phát hiện root/jailbreak, mock-location (jail-monkey), chống debug/emulator, kiểm tra proxy/VPN
5. Bảo vệ người dùng MFA, OTP, Biometric cho giao dịch quan trọng

AES Mã hóa đối xứng

Mã hóa payload trước khi gửi server, giải mã ở Back-end bằng cùng secret key — bảo vệ dữ liệu khi truyền tải.

crypto.ts
const encrypted = CryptoJS.AES.encrypt(
  JSON.stringify(data), secretKey
).toString();

RSA Mã hóa bất đối xứng

Public Key dùng để mã hóa / xác minh, chỉ Private Key (lưu trong Keychain/Keystore) mới giải mã được — dùng cho chữ ký số & trao đổi dữ liệu nhạy cảm.

  • Ai cũng biết Public Key, chỉ chủ Private Key giải mã được
  • Chữ ký số ký bằng Private Key để xác thực nguồn gốc & tính toàn vẹn

ProGuard Obfuscation cho Android Release

minifyEnabled true + shrinkResources true trong build.gradle, kết hợp custom rules trong proguard-rules.pro để bảo vệ class React Native, Hermes, Retrofit/OkHttp.

JWT Header · Payload · Signature

Header chứa thuật toán mã hóa, Payload chứa claims (quyền, expire time), Signature dùng secret-key trên server để xác thực tính toàn vẹn.

03

Core Concepts & Architecture

Nền tảng JS/TS, mô hình luồng (threading) trong React Native, và tư duy thiết kế hướng SOLID — những gì giúp tối ưu & bảo mật ở trên thực thi đúng cách.

JS Runtime Event Loop & Microtask / Macrotask

Hiểu rõ thứ tự thực thi Promise.then / queueMicrotask (microtask) trước setTimeout / setInterval (macrotask) — tránh block event loop bằng cách chia nhỏ task nặng.

event-loop.ts
console.log("1");
setTimeout(() => console.log("2"));
Promise.resolve().then(() => console.log("3"));
console.log("4");
// ⇒ 1 4 3 2

Threading JS · UI · Native Modules

3 luồng độc lập phối hợp qua React Native Bridge:

  • JS Thread — logic, state, render request (Hermes/JSC)
  • UI Thread — vẽ & cập nhật giao diện native
  • Native Modules Thread — tác vụ nặng (ảnh, file, mã hóa) chạy background, không chặn UI/JS

Async Promise.all / allSettled / race / any

MethodĐợi tất cả?Trả về
Promise.allCó, fail-fastMảng kết quả
Promise.allSettledCó, không failMảng status/value/reason
Promise.raceKhôngKết quả đầu tiên hoàn thành
Promise.anyKhôngKết quả thành công đầu tiên

OOP / SOLID Tư duy thiết kế dự án lớn

Áp dụng 4 tính chất OOP (đa hình, kế thừa, đóng gói, trừu tượng) và nguyên tắc SOLID khi thiết kế component & props — dễ mở rộng, dễ maintain ở dự án quy mô 10-30 người.

Native Bridge iOS & Android Native Modules

Triển khai Native Module giao tiếp 2 chiều qua Promise & Event:

  • iOS: RCTBridgeModule, RCT_EXPORT_METHOD
  • Android: extends ReactContextBaseJavaModule, @ReactMethod, đăng ký qua ReactPackage

Realtime SSE vs WebSocket

SSE (qua EventSource) — kết nối 1 chiều từ server, độ trễ thấp, độ phức tạp thấp hơn Socket. Dùng cho các luồng cập nhật dữ liệu thị trường realtime trong app Fintech.
